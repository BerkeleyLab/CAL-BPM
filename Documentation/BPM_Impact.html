How to Create a Beam Position Monitor FPGA Image

1 – Preliminary – Ensure FPGA Build Date is Correct

Remember to update the firmware build date:
cd ...src/BPM/head/BPM.srcs/sources_1/edk
sh setFirmwareDate.sh
startXilinx.sh planahead
Generate Bitstream.
File→'Export'→'Export Hardware for SDK'.
Check all boxes – Include Bitstream, Export Hardware, Launch SDK.

2 – Preliminary – Create the bootloader image

  1. In SDK: Xilinx Tools→Program FPGA
  2. Select the bootloader executable:
  3. Click the Program button.
    Don't worry about an error window saying that the FPGA can't be written – the download.bit file has been created before this occurs.
  4. Assuming that the steps below have been performed once the boot image can be produced by running a couple of simple scripts:
    sh startXilinx.sh bash
    cd src/BPM/head/BPM.sdk/SDK/SDK_Export/Impact
    sh mergeBoot.sh
    sh createBin.sh
  5. The resulting image can then be transferred to the BPM flash memory using TFTP.

3 – Configure the Impact project

This step need be performed only once since from then on the BPM.ipf file can be used.
  1. Start Impact:
    sh startXilinx.sh impact
  2. Click No when asked to automatically load the last saved project:
  3. Click Yes to automatically create and save a project:
  4. Select the Prepare a PROM file option and click OK:
  5. Configure a:
    BPI Flash – Single FPGA
    Virtex 6, 128M
    Ouput file name BPM.bin, in src/BPM/head/BPM.sdk/SDK/SDK_Export/Impact, BIN format (Swap Bits ON), x16
    Click Ok:
  6. Click OK to start adding device file:
  7. Select the download.bit file created in section 2:
  8. Click No when asked to add another device file:
  9. Click OK:
  10. In the data file assignment window that appears, click OK:
  11. Save the Impact project in src/BPM/head/BPM.sdk/SDK/SDK_Export/Impact/BPM.ipf

4 – Use the Impact project to create the flash image

Start Impact:

sh startXilinx.sh bash
cd src/BPM/head/BPM.sdk/SDK/SDK_Export/Impact
impact -ipf BPM.ipf

This will bring up the window shown below.
Double click the Generate File... entry and wait for the Generate Succeeded message to appear.
Quit Impact without saving.
The directory will now contain a BPM.bin file suitable for transferring to the BPM flash.
