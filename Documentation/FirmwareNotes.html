Overview

The following figure shows the main components of the beam position monitor firmware.



Data and clock values from each ADC are received by a register clocked by an output from an FPGA multi-clock module synchronized to the clock generator chip on the AFE.  Reading the individual ADC clock signals allows the startup code to adjust the phase of the input register clock to place the sampling instant as far as possible from data line transitions.  The peak value of each of the ADC readings over the event system heartbeat interval is passed to the MicroBlaze slow acquisition block.

The ADC readings are passed to a waveform recorder and to the preliminary processing block which measures the signal strength from each of the signal pickup buttons.  The preliminary processing block provides the signal strengths at the RF frequency and at three three different update intervals:
  • A single turn (or a single-shot for transport line applications)
  • ~100 µs (the ‘fast acquisition’ interval)
  • ~100 ms (the ‘slow acquisition’ interval)

The fast and slow acquisition intervals are not exactly 100 µs and 100 ms respectively since each interval must be an integral multiple of turn intervals.

The preliminary processing block also provides signal strengths at two pilot tone frequencies, one half the ring revolution frequency lower than the ring RF frequency and one half the ring revolution frequency higher than the ring RF frequency, at a programmable update interval.

    The three sets of button RF signal strength measurements are passed to the position calculation block.  The signal strengths of all three frequencies measured over the slow acquisition interval are also passed directly to the embedded processor.  The position calculation block computes the beam X and Y positions in nanometers, a skew factor (Q) and the sum of the button signals (S).  The current version of the firmware applies neither offset nor pincushioning corrections.  The values measured over the turn interval are passed to a waveform recorder.  The values measured over the fast acquisition interval are passed to another waveform recorder,  to the RMS calculation block, and are filtered and passed to a cell communication block.  The cell communication block forwards the filtered fast acquisition position readings to the monitor’s clockwise and counterclockwise neighbors.  The RMS calculation block computes narrowband and wideband beam motion.  The values measured over the slow acquisition interval are passed to the embedded processor.

    The embedded processor block is a single processor MicroBlaze system which acts as a network-attached device communicating with an EPICS IOC.  The system includes an embedded event receiver which provides the required timing and synchronization signals.

    Timing and Synchronization

    The timing system reference clock recovered by the BPM event receiver firmware provides the base for all acquisition and signal processing operations.  The reference clock is 1/4 the ring RF frequency.  A set of counters divides the EVR reference clock to provide a reference to the AFE PLL and to generate the slow and fast acquisition timing markers.   The arrival of a ‘heartbeat’ event (event number 122) reloads all the counters.  If the counter was about to reload anyways the status of that counter is set to ‘synchronized’, otherwise to ‘unsynchronized’.

    The reference clock is divided by:
    • 82 (ALS storage ring and transport lines) or 125 (booster ring) to provide the reference for the AFE PLL which multiplies it by 77 (ALS storage ring or transport lines) or 116 (booster ring) to produce the ADC sampling clock.
    • 12,464 (82 timing system clocks per turn × 152 turns) for the storage ring or by 11,875 (31.25 timing system clocks per turn × 380 turns) for the booster ring to produce the fast acquisition marker.  The fast acquisition rate is approximately 10 kHz  for the storage ring and about 10.5 kHz for the booster because of the need to acquire an integral number of turns per fast acquisition update and per slow acquisition update.  There is no requirement for an integral number of fast acquisitions per slow acquisition.
    • 12,464,000 (152,000 storage ring turns or 398,840 booster ring turns) to produce the slow acquisition marker.  The slow acquisition rate of approximately 10 Hz is the same for all BPMs.
    The heartbeat event is transmitted by the timing system event generator every 124,640,000 reference clock cycles which is about 0.998 seconds.

    ADC Clocks

    Four outputs from an Analog Devices AD9510 clock generator provide the sampling clocks for the analog to digital converters.  Another AD9510 output is taken to the FPGA to provide a clock for the logic that operates at the ADC sampling frequency.  An IODELAYE1/IODELAYCTRL combination in the FPGA provides a programmable delay, adjustable in ~78 ps steps, which is adjusted to meet two constraints:
    1. On power-up the delay is scanned through its entire range to find the point at which the rising edge of the FPGA ADC clock is as far as possible from the falling edge of the clock signals from the individual ADCs.   This ensures that the data lines from the ADCs are sampled with no chance of race conditions or metastability.
    2. During BPM operation a state machine verifies that the same number of FPGA ADC clocks are present between each rising edge of the slow-acquisition (SA) marker from the event system. If the number of clocks differs between rising SA marker edges it indicates that the rising edge of the SA marker, which is in the event receiver clock domain, is too close to the rising edge of the FPGA ADC clock.  The state machine then adjusts the ADC clock delay to move the FPGA ADC clock away from the rising edge of the SA marker.

    ADC Readout

    The ADC readout logic checks for ADC clipping, records the peak ADC reading and provides delays and triggers as shown in the following figure.



    The raw ADC samples have any D.C. component removed if RMS signal processing is enabled and are delayed by 12 ADC clock cycles before being passed to the preliminary processing block.  The absolute value of the ADC value, with any D.C. component removed is compared to a threshold and the "ADC self trigger" line is asserted when any channel is above the specified threshold.   A reading above threshold also starts a counter which when non-zero asserts the "use this sample" indication to the RMS signal processing logic.  The combination of the delay block and this counter result in the "Use this sample" line being true for several samples before any ADC value exceeds the specified threshold and after the last sample for which any channel exceeds the threshold.


    Preliminary Processing

    The following block diagram shows the signal processing operations applied to the data from each ADC channel.  The synchronous demodulation multipliers and accumulators operate in the FPGA ADC clock domain.  The CORDIC block and all other logic operates in the system clock domain.

    There are three synchronous demodulators for each ADC channel.  One demodulator picks out the frequency component matching the RF reference.  The other demodulators pick out the frequency component of the pilot tones typically located about 1/2 the storage ring revolution frequency above and below the RF reference.  Two accumulators for the RF demodulator provide averaging over a single turn and over some small number of turns, typically two.  Pilot tone accumulation also takes place over the same small number of turns.  The signal amplitudes obtained from the latter accumulators are passed through CIC decimation filters to provide RF and pilot tone signal amplitudes at the fast acquisition rate and RF signal amplitudes at the slow acquisition rates.  The demodulator local oscillator coefficients are set from tables stored in flash memory.  The length of these tables sets the number of samples accumulated in the synchronous demodulation.  For proper operation the FA and SA CIC filter decimation factors must be integers (i.e. Frev/FFA/N must be an integer).

    A single CORDIC block is shared by all demodulation frequencies and intervals and by all ADC channels.  The block converts the demodulated I/Q signals to magnitude/phase coordinates.  The block is fully pipelined and can provide a new output value every 10 ns.

    To make the best use of the fixed-point arithmetic used in the CORDIC and all subsequent blocks the multiplier outputs can be shifted upwards before being applied to the accumulators.  Shifting upwards by too little results in excessive quantization errors in the computations and shifting upwards by too much causes accumulation overflows to occur.

    For BPMs installed outside the storage ring both pilot tone local oscillators are the same as the RF reference.  The pilot tone is activated and measured at times when beam is not present.  For single-pass BPMs the turn-by-turn signal processing chain is used to compute the button beam signal magnitudes and the results sent to the turn-by-turn and slow acquisition position computation chains.  The fast acquisition beam signal processing chain is unused.

    The gain compensation factors can be adjusted automatically as described in the ‘Auto Trim’ section of the EPICS documentation.






    RMS button signal processing

    For sparse fill patterns or for transport line applications the preliminary processing firmware can be placed in a mode which computes the RMS of the button signals rather than the magnitude of a single component of the discrete Fourier transform of the signals.   In this mode the local oscillator cos(ωRFt)\cos(\omega_{\text{RF}}t) input to the uppermost multiplier in the figure is replaced with the ADC reading itself (thus forming the square of the ADC reading),  the accumulated squares are passed through a square root block before being taken to the CORDIC 'x' input, and the CORDIC 'y' input is set to 0.  The "Use this sample" line from the ADC readout block is used as an enable line o the accumulators.  Although the RF local oscillator values are unused in this mode of operation the table of values must still be present since the table length sets the maximum number of samples to be used in the RMS computation.  The actual number of samples may be smaller since only those samples for which the 'Use this sample' line is true will be added to the accumulators.

    CIC Decimation Filters

    Decimation from turn-by-turn to fast acquisition and from fast acquisition to slow acquisition rates is achieved by CIC (cascaded integrator-comb) filters.  The transfer function for an N stage CIC filter with decimation factor R is:

    H(z)=(∑k=0R-1z-k)NH(z)=\left( \sum_{k=0}^{R-1} z^{-k} \right)^N

    For the turn-by-turn to FA decimation filter R is depends on the number of turns used do accumulate the synchronous demodulation products and for the FA to SA decimation filter R is 1000.  For both filters N is 2.

A trigger is generated when the filtered sum falls below seven-eighths of the filtered sum 64 turns earlier minus a ‘Threshold Adjust’ value set by an EPICS record.



    Position Calculation

    A block diagram of the position computation module is shown below.  The signals from the preliminary processing module are passed through a group of 3-to-1 multiplexers to select the demodulated button signal magnitudes for one of the three update rates.  The magnitude signals are then passed through a second group of multiplexers which allows the difference of any two buttons or the difference between the sum of any pair of buttons to be calculated.  This allows complete flexibility in button assignment as well as providing for buttons at 0º or 45º rotation.  The difference value is then divided by the sum of the four button signals.  The quotient is multiplied by a calibration factor to produce a 32 bit position in units of nanometers.  A final group of registers holds the values for each update rate.  All difference and sum values are processed by a single divider and multiplier.  The results are demultiplexed and stored in the appropriate output latch.

    The ten input selection lines for the second group of multiplexers (two 2-bit values and two 3-bit values) for the three (X, Y, Q) calculations are set by an I/O register from the microBlaze processor.   The values depend on the button mapping and rotation values contained in the system parameters.  Future expansion may allow the register to be written by an EPICS longout record for applications requiring non-standard computations.

    The position calculation block can accept a new set of signals every 15 clock cycles.  This is sufficient to handle the booster ring which produces a new set of turn-by-turn signals every 25 clock cycles since turn-by-turn data take priority over fast-acquisition or slow-acquisition values.

    Cell Data Stream Filtering

    The X and Y 'fast acquisition' beam position values are passed through 300-tap FIR filters before being sent to the cell communication block.  Each plane has its own filter with a set of 300 programmable coefficients.  The default filter coefficients provide a second order Butterworth low pass response with 60 Hz cutoff.  The filters add about 2 µs of latency.


    Cell Communication

    High speed serial links to clockwise and counterclockwise neighbors are based on the Xilinx Aurora 8B/10B protocol with parameters:

    •     3.125 Gb/s
    •     1 lane
    •     4 bytes per lane
    •     CRC enabled

    After calculating fast acquisition values the BPM produces a packet which it transmits on both CW and CCW links.  Each BPM also forwards packets received from the CW link to the CCW link and vice versa.  Thus, under normal conditions, the cell controller receives a packet from each BPM on each link at the fast acquisition rate.  Each packet consists of five 32-bit words:

    1. A header containing the value 0xA5BE in the most significant 16 bits, 7 padding (0) bits and, in the least significant 9 bits, the fast orbit feedback index of the BPM providing this packet.
    2. The beam X position, in nm.
    3. The beam Y position, in nm.
    4. The sum (S) value in the least significant 30 bits.  The next-to-most significant bit is set if any of the ADCs were clipping when the button values were acquired.  If a packet is received with a CRC fault the most significant bit of this word will be set in the forwarded data.  Packets received without CRC faults are simply forwarded in their entirety.  This ensures that the cell controller can detect the presence of invalid data.
    5. The Aurora CRC word.



    Beam Motion RMS Calculation

    The RMS value of the X and Y beam motion in two frequency ranges is computed as shown in the following figure.  The computed values are included in the slow acquisition data packets sent to the IOC.  A first order IIR high pass filter removes the DC component of the input values.  The frequencies to be included in the narrow band measurement are selected by a FIR filter with parameters:

    • Pass band 0 to 200 Hz
    • Stop band 400 to 5000 Hz
    • Pass band ripple 0.67 dB
    • Stop band attenuation 60 dB or better

    The first order IIR low pass filters between the square and square root operations simulate the ‘mean’ portion of the RMS calculation. 

    Raw ADC value transmission

    Raw ADC values and a timing synchronization signal are transmitted as a two-lane 64B/66B Aurora link on front panel SFP slots 5 and 6.  The least significant 64 bits of the resulting 128 bit AXI stream contain values from ADC sample N and the most significant contain values from ADC sample N+1.  The 14 bit ADC values are left-aligned in the four sixteen bit fields of a 64 bit sample with ADC 0 in the least significant 16 bits and ADC 3 in the most significant 16 bits.  The least significant bit of the ADC 3 field (i.e. bit 48 of the 64 bit sample) is the turn marker.
     

    MicroBlaze System

    The components making up the MicroBlaze embedded system are shown below.


    The processor is configured with most of the performance options enabled.  The system clock rate is 100 MHz.

    • 32 kB instruction cache with speculative fetch and 8 ‘cache victims’
    • 32 kB ‘write back’ data cache with 8 ‘cache victims’
    • Branch target cache
    • Barrel shifter
    • 64-bit multiplier
    • Integer divide

    A 128 bit AXI bus provides multiple ports to the SDRAM.  The 128 bit width allows two ADC samples (16 bits per channel, 4 channels per sample) to be transferred from the waveform recorder AXI master to the SDRAM in each cycle of the 100 MHz system clock which is necessary given that the ADC clock is faster than the system clock.  The AXI masters connected to the SDRAM are:

    • A read-only connection from the processor instruction port.
    • A read/write connection from the processor data port.
    • Three DMA connections from the Ethernet controller.
    • A write-only DMA connection from the BPM ADC value waveform recorder.
    • A write-only DMA connection from the BPM turn-by-turn waveform recorder.
    • A write-only DMA connection from the BPM fast acquisition (~10 kHz) waveform recorder.

    An AXI-Lite bus provides connections to:

    • The 128 Mbyte flash memory used to hold the FPGA firmware, the MicroBlaze software, and the system configuration tables.
    • The debug module used to controller the processor during JTAG operation.
    • The interrupt controller.  The Ethernet controller is the only component capable of requesting an interrupt.
    • The Ethernet controller and DMA block.
    • The system monitor block.  This allows the MicroBlaze to monitor FPGA voltages and temperature.
    • The ICAP block that the MicroBlaze uses to force an FPGA reset operation on request.
    • The console serial line.
    • The embedded event receiver.
    • A general-purpose I/O block providing 128 write strobes and 128 32-bit inputs used to communicate with the other custom firmware of the BPM.

    Unpacking and building an archived system

    Some source files are machine generated.  The scripts to produce these are included in the archived project but need to be run before attempting to build the firmware.

    To unpack an archived project and run the scripts:

    unzip ../BPM.pa.zip
    cd BPM/BPM.srcs/sources_1/edk/bpm/
    python ../../imports/bpm/createLocalOscillatorTables.py
    bash ../../imports/bpm/setFirmwareDate.sh
    cd ../../../../BPM.sdk/SDK/SDK_Export/BPM_APP/src/
    sh createVerilogIDX.sh
    startXilinx.sh planAhead

    A number of critical warnings about missing constraints will be reported when building the system.  These appear the be the result of using the soft Tri-Mode Ethernet firmware and come from code automatically generated by the Xilinx tools.  I‘ve found no way to eliminate the warnings but they appear to be benign.  There are lots of other non-critical warnings as well.  I‘ve checked these and confirmed that all are benign at the time this note was written.  Many are simply the result of using generic integer literals and several others are produced from Xilinx-generated firmware.

    Adapting firmware to different environments

    Make appropriate changes to the system parameters to set the ADC sampling clock and FA/SA/Heartbeat rates.  New local oscillator coefficient tables must also be generated.  Large rings may need to have the local oscillator coefficients divided by a constant to avoid overflowing the turn-by-turn synchronous demodulator accumulators.
