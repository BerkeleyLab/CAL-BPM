How to configure a new Beam Position Monitor FPGA

1 – Preliminary

  1. Connect a programming pod to the JTAG port of the BPM
  2. (Optional) Connect a serial cable to the console port of the BPM and connect to it using a terminal emulator set at 115200-8N1.
  3. Prepare the BPM firmware and software.


2 – Use SDK to download firmware and software

These steps can be skipped if the BPM is already capable of booting from its flash memory.

  1. Start SDK:
    sh ~wenorum/bin/startXilinx.sh xsdk
  2. Choose the BPM development workspace.
    ~wenorum/src/BPM/head/BPM.sdk/SDK/SDK_Export
    Write access is required so it may be necessary to copy the development workspace to your directory tree and then browse to, and use,  that copy.
  3. Xilinx Tools → Program FPGA
  4. Select the BPM firmware and the bootloader.
  5. Run the BPM application.  Select BPM_APP in the left pane then click the green triangle.
  6. The BPM application should start.  On the console you will see a start-up message and some complaints about corrupt parameter and attenuation tables:

3 – Upload the firmware, software, parameters and attenuation tables to flash memory

The BPM provides a TFTP file server so updating the flash memory is simply a matter of uploading the appropriate files to the BPM.  This is very straightforward if the BPM is already on the network.  The procedure is a little more complicated for a brand new BPM or one with corrupt parameters since it will have assigned itself default Ethernet and IP addresses as shown above.  You can either add a route to this address or temporarily connect the BPM to a laptop computer with Ethernet address set to a 192.168.1.0/24 address.   For the examples shown below the laptop address was set to 192.168.1.10 with net mask 255.255.255.0.   An alternate possibility is to use the console MAC and NET commands to set the network parameters in flash memory, connect the BPM to the netwrok and then repeat the above section from step 5.

Note that the files must be transferred in 'binary' mode.  The file name comparison in the BPM ignores case and any characters after the initial part of the name, so for example, 'Parameters-BPM123-2014-09-21.csv' will refer to the same contents as 'Parameters.csv'.
  1. Upload BPM firmware.
    210 > cd ~wenorum/src/BPM/head/BPM.sdk/SDK/SDK_Export/Impact
    211 > ls
    BPM.bin        BPM.ipf        BPM_xdb        sendBIN.sh
    BPM.cfi        BPM.prm        createBIN.sh
    wenorum@enorum 213> tftp 192.168.1.20
    tftp> bin
    tftp> put BPM.bin
    Sent 9232444 bytes in 75.3 seconds
    tftp> quit

  2. Upload BPM software.
    214 > cd ~wenorum/src/BPM/head/BPM.sdk/SDK/SDK_Export/BPM_APP/Release
    215 > ls
     BPM_APP.elf             BPM_APP.srec    sources.mk
     BPM_APP.elf.elfcheck    makefile        src
     BPM_APP.elf.size        objects.mk
    216 > tftp 192.168.1.20
    tftp> bin
    tftp> put BPM_APP.srec
    Sent 558112 bytes in 9.6 seconds
    tftp> quit

  3. Set parameters.  If you don't have a "Parameters.csv" already an easy approach is to first download the existing file, use an editor or spreadsheet program to make the desired changes and then upload the modified file.
    220 > cd ~/tmp
    221 > tftp 192.168.1.20
    tftp> bin
    tftp> get Parameters.csv
    Received 225 bytes in 0.0 seconds
    tftp> quit
    222 > <<< Edit Parameters.csv with text editor or spreadsheet program >>>
    223 > tftp 192.168.1.20
    tftp> bin
    tftp> put Parameters.csv
    Sent 226 bytes in 0.1 seconds
    tftp> quit

  4. (Optional) Set the attenuation table in the same way that the parameters were set, but with file name "Attenuation.csv" in place of "Parameters.csv".

Power cycle the BPM and confirm that it starts up and finds the updated parameters and attenuation values.
